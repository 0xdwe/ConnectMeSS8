## Findings

### Finding 1 — BLOCKER: `firstName` string interpolation is escaped

**File:** `lib/src/features/ai_update_screen.dart`, line 182

**Problem:** The `onClassifierPassed` callback sets `_loadingLabel` to:
```dart
"Reading what you've shared with \$firstName…"
```

The `\$` is a Dart escape sequence for a literal dollar sign. This means the string is literally `"Reading what you've shared with $firstName…"` — the contact's first name is never interpolated.

**Evidence:**
- `dart analyze` reports: `warning - lib/src/features/ai_update_screen.dart:162:11 - The value of the local variable 'firstName' isn't used.`
- The `firstName` variable is computed at line 160-161 but never referenced anywhere.

**Fix:** Remove the backslash:
```dart
"Reading what you've shared with $firstName…"
```

**Severity:** BLOCKER — user-visible copy is wrong (shows `$firstName` literally).

### Finding 3 — NOTE: Label transition test doesn't assert the transition

**File:** `test/features/ai_update_loading_test.dart`, lines 292-310

**Problem:** The test `'loading label transitions from Checking to Reading'` only asserts the initial label ("Checking your input…") and then drains timers. It does not assert that the label changes to "Reading what you've shared with…". The comment says callback wiring is "proven by llm_ai_update_test.dart", but that only proves the adapter calls the callback — it does not prove the widget state updates correctly in response.

**Severity:** NOTE — not a blocker because the callback wiring is correct (verified in `llm_ai_update_test.dart` L1196-1230), but the widget test should be hardened to assert `find.textContaining('Reading')` after a 100ms pump (the MockAiUpdate fires `onClassifierPassed` after 50ms). This gap allowed Finding 1 to go undetected.
