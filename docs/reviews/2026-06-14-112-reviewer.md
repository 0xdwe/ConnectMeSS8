## Findings

### NICE-TO-FIX

**N1 — Redundant `.timeout()` at classifier call site** (`lib/src/ai/llm_ai_update.dart:298-305`)

The classifier wraps the result of `effectiveGemini(...)` with `.timeout(kLlmAiUpdateClassifierTimeout)`, and `_defaultGeminiGenerateContent` already applies the same timeout internally via `generateContent(contents).timeout(timeout)`. The outer `.timeout()` is a defense-in-depth safety net for injected functions that ignore the `timeout` parameter, which is sound. But the two-timeout pattern is slightly confusing to read. Consider adding a brief comment explaining that the outer `.timeout()` is a belt-and-suspenders guard for test-injected functions.

**N2 — `onClassifierPassed` throw is swallowed by fail-open** (`lib/src/ai/llm_ai_update.dart:311,318`)

`onClassifierPassed?.call()` sits inside the `try` block. If the callback throws, the `catch (e)` at line 318 absorbs it silently (only `debugPrint`). This is intentional — a loading-label update should never block an AI update — but it means a developer error in the callback (e.g., an NPE) is invisible beyond log output. Consider wrapping in its own try-catch that still fail-opens but prints the callback error distinctly from the classifier error.

**N3 — Cancellation test return statement is dead code** (`test/state/ai/llm_ai_update_test.dart:1387`)

```dart
await Completer<void>().future;
return '{"isRelevant": true}';
```

The `await` on a never-completing future means the `return` is unreachable. The test is correct (cancel wins the race), but the dead `return` could mislead future readers. Replace with `// never reached — cancel wins the race` comment and drop the `return`, or assert-fail.

### NOTE

**N4 — Pre-existing test noise from classifier fail-open** (test output line `+330`)

The pre-existing `all-images-failed-AND-no-text → no hard fail` test constructs `LlmAiUpdate` with `firebaseAi: null` and no `geminiGenerateContent`. Before #112, this test hit the SDK guard `StateError`. After #112, the classifier fires first, fails open (because `_defaultGeminiGenerateContent` also throws `StateError` for null `firebaseAi`), then the main call reaches the same `StateError`. The test assertion (`throwsA(isA<StateError>())`) still passes. The only observable difference is a `debugPrint` line in test output:

```
LlmAiUpdate: relevance classifier failed open: Bad state: LlmAiUpdate.run reached the SDK call site...
```

This is harmless and expected — zero assertions changed.

**N5 — `effectiveGemini` local variable** (`lib/src/ai/llm_ai_update.dart:287`)

The getter `_effectiveGeminiGenerateContent` is evaluated once into a local `effectiveGemini`, avoiding repeated resolution. The local is scoped to `run()` body, which is appropriate since the resolver is cheap (null-check). No issue.

## Architecture check

- **Function injection pattern**: `GeminiGenerateContentFn` typedef mirrors the existing `attachmentPreparer` pattern exactly (`lib/src/ai/llm_ai_update.dart:60-68`). The `geminiGenerateContent` parameter is nullable with a private getter that resolves to `_defaultGeminiGenerateContent` when absent (lines 107-109, 408-409). Matches convention.

- **Cancel-token race**: Classifier call is wrapped in `raceWithCancel` (line 298), same as the main call (line 354). The error-absorbing listener on the orphan future (line 264) prevents unhandled-async-error noise when cancel wins the race.

- **No image bytes to classifier**: Only `attachments.isNotEmpty ? 'yes' : 'no'` is sent (line 295), not image data. Verified against AC.

- **No MemoryDocument to classifier**: Classifier prompt uses `contact.name`, `contact.category`, `userInput`, and the bool flag only. No memory doc. Verified against AC.

- **Anti-shame guardrail**: Prompt explicitly states "Never mention numeric day counts, time elapsed, or anything that could shame the user" (`llm_ai_update_prompt.dart:49-50`). Matches the codebase guardrail from AGENTS.md.

- **No dead code / duplicate typedefs**: `GeminiGenerateContentFn` is the only new typedef and is used in one place (constructor parameter). No duplication with existing typedefs.
