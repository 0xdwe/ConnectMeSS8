# Foundation: design token system + tri-state theme

Labels: enhancement, ready-for-agent, foundation

> *This was generated by AI during triage.*

## AI Triage Brief

**Category**: enhancement. Foundational refactor introducing semantic design tokens and a tri-state theme mode (system / light / dark). No visible UX change beyond colors becoming consistent.

**State**: ready-for-agent. Token names, hex values, and roles are pinned in `DESIGN.md`. Mechanical migration.

**Codebase notes**:
- Theme lives in `lib/src/theme/app_theme.dart`. Currently exposes raw colors as `static const` (`AppTheme.moss`, `AppTheme.sage`, `AppTheme.clay`, etc.).
- `AppState.darkMode: bool` in `lib/src/state/app_state.dart` and `AppController.setDarkMode(bool)` is the existing toggle.
- Theme is selected in `lib/src/app/connect_me_app.dart` via `themeMode: appState.darkMode ? ThemeMode.dark : ThemeMode.light`.
- Hard-coded hex appears in 14+ files: search `0xFF` and `Color(0x` to find them. Big offenders: `crm_widgets.dart`, every `tabs/*.dart`, `contact_profile_screen.dart`, `shell_screen.dart`.

**Implementation notes for the agent**:
- Add `AppTokens extends ThemeExtension<AppTokens>` in `lib/src/theme/app_tokens.dart`. Fields per `DESIGN.md` color tables.
- Two factory constructors: `AppTokens.light()` and `AppTokens.dark()` with values from `DESIGN.md`.
- Wire `AppTokens` into `ThemeData.extensions` in both light/dark `AppTheme.data()`. Provide a `BuildContext.tokens` extension for ergonomic reads (`Theme.of(context).extension<AppTokens>()!`).
- Replace `enum-less darkMode: bool` with `enum AppThemeMode { system, light, dark }` on `AppState`. Default = `system`. Update `copyWith`, `seeded`, `signOut`.
- Replace `setDarkMode(bool)` on `AppController` with `setThemeMode(AppThemeMode mode)`. Keep `setDarkMode(bool)` as a deprecated shim mapping `true → dark`, `false → light` if any callers can't be updated in this PR.
- In `ConnectMeApp`, resolve effective `ThemeMode` from `themeMode`: `system → ThemeMode.system`, `light → ThemeMode.light`, `dark → ThemeMode.dark`.
- Update `theme_modal.dart` to a three-radio picker.
- Migrate `AppSurface`, `AppHeader`, `CardBox`, every widget that uses `AppTheme.moss/sage/clay/etc.` or raw hex, to read `tokens.primary / tokens.surface / tokens.surfaceRaised / tokens.ink / etc.`
- DO NOT change typography in this PR (separate issue 010). DO NOT change user-level XP fields (separate issue 011).
- `Colors.black54`, `Colors.black38`, `Colors.black` for text → `tokens.inkMuted` / `tokens.inkSubtle` / `tokens.ink`. The existing `Color(0xFF667085)`, `Color(0xFF4B5563)`, `Color(0xFF374151)` → `tokens.inkMuted`.
- Tests: existing widget tests should keep passing (no visible UX change). Add `test/theme/app_tokens_test.dart` asserting that `AppTokens.light().primary` matches the documented value, and that `AppController.setThemeMode(AppThemeMode.system)` followed by a `MediaQuery.of(context).platformBrightness` of `dark` results in `Theme.of(context).brightness == Brightness.dark`.

## What to build

A single source of truth for color in ConnectMe. Every widget reads named tokens, not raw hex. Theme mode is tri-state, system-following by default.
