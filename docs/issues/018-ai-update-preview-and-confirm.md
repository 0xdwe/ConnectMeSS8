# AI Update: preview-and-confirm flow

Labels: enhancement, ready-for-agent, surfaces

> *This was generated by AI during triage.*

## AI Triage Brief

**Category**: enhancement. Refactors the AI update flow so AI never silently mutates state. User submits free text → AI parses → preview screen shows proposed `CrmInteraction`s for editing → user confirms or cancels.

**State**: ready-for-agent. Flow specified in `PRODUCT.md → Design Principle 4` and `DESIGN.md → Components → AI preview`.

**Codebase notes**:
- `lib/src/features/ai_update_screen.dart`: input → submit → `runAiUpdate()` → app pops back. No preview, no confirmation.
- `lib/src/state/app_state.dart` `runAiUpdate(contactId, input, attachments)`: awaits `aiUpdateServiceProvider` then mutates `state.interactions`, `state.connections`, and sets `state.lastAiSummary`.
- `lib/src/ai/ai_update_service.dart` `AiUpdateService.categorizeAndUpdate(...)` returns `AiUpdateResult { summary, contactId, interactions, nextStep }`.
- The mock implementation builds interactions from input keywords.

**Implementation notes for the agent**:
- New screen state machine in `AiUpdateScreen`:
  - `inputting` (default): the existing form (text + attach + add image).
  - `previewing`: show the parsed `AiUpdateResult` as editable cards. Confirm + Cancel.
  - `saving`: brief loading state.
  - `done`: pop and show a snackbar "Logged 3 updates with Sarah" with an "Undo" action.
- Refactor `runAiUpdate` on `AppController`:
  - Split into two methods:
    - `Future<AiUpdateResult> previewAiUpdate(...)`: calls service, returns the result without mutating state.
    - `void commitAiUpdate(AiUpdateResult result)`: applies the mutations (interactions/contacts/lastAiSummary) atomically.
- Update `AiUpdateScreen` flow:
  1. User taps "Update Connection" (existing `run-ai-button`). Calls `previewAiUpdate(...)`. Sets `state = previewing` with the result.
  2. Preview UI:
     - Header "Here's what I found" (`AppTypography.h2()`).
     - For each `CrmInteraction` in `result.interactions`, an editable card:
       - Contact match (avatar + name) — tappable to swap contact (out of scope for this PR; use AI's match, no swap UI).
       - Type chip (read-only for this PR).
       - Title `TextField`, prefilled with `interaction.title`.
       - Note `TextField` multiline, prefilled with `interaction.note`.
       - Date row showing `DateFormat.yMMMd()`.
       - Small `Container` with ✨ icon + "AI suggested" caption.
     - Footer row: `FilledButton("Save these (${count})")` + `TextButton("Cancel")`.
  3. Save: applies user edits to the in-memory `AiUpdateResult` (rebuild `CrmInteraction`s with edited title/note), calls `commitAiUpdate`, pops, shows a snackbar with Undo (`SnackBarAction(label: 'Undo', onPressed: () => state.removeInteractionsByIds(...))`).
  4. Cancel: discards, returns to `inputting` mode (or pops if user prefers — wire a back button + a "Edit again" affordance).
- Add an "✨ AI suggested" tag on each saved `CrmInteraction`. The model `CrmInteraction` doesn't currently have a source field — add `final InteractionSource source` (default `manual`, set `aiSuggested` for AI-saved). Render the chip on history list rows in `contact_profile_screen.dart` (issue 017 may have updated that file recently).
- The Update Connection AI flow is now reachable from two places: per-contact (`/ai-update/:id`) and globally via the + sheet (issue 014). Make sure both entry points work.
- DO NOT add the stagger animation in this PR. That's issue 022.
- Tests:
  - Existing tests that pump `runAiUpdate` directly need updating (it's split into two methods).
  - New widget test: input → tap Update Connection → preview shows N cards → tap Save → snackbar shows → Undo removes the saved interactions.
  - Cancel from preview returns to inputting state.

## What to build

A two-step AI flow. AI parses to a preview the user can edit, not state the app silently changes.

## Acceptance criteria

- [ ] `AppController.previewAiUpdate(...)` exists and returns `AiUpdateResult` without mutating state.
- [ ] `AppController.commitAiUpdate(AiUpdateResult)` exists and applies mutations atomically.
- [ ] `AiUpdateScreen` has explicit `inputting`/`previewing`/`saving` states.
- [ ] Preview screen shows editable title and note fields per parsed interaction.
- [ ] Preview includes "Save these (N)" primary button and "Cancel" text button.
- [ ] Save commits, pops, shows a snackbar with an Undo action that removes the saved interactions.
- [ ] Cancel discards, returns to inputting.
- [ ] `CrmInteraction` model has a `source` field (default manual; AI-saved set to aiSuggested).
- [ ] AI-saved interactions render with a small ✨ tag in the contact profile history.
- [ ] `runAiUpdate` is removed (or kept as a thin wrapper that previews + commits, only for tests).
- [ ] Tests updated. New tests cover preview → save and preview → cancel.
